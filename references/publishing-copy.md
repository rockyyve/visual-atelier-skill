# Publishing Copy

Write publish-ready Xiaohongshu copy with the images. Do not leave it as a later step.

## Required Sections

The companion Markdown document must include:

- Recommended title.
- 6-10 alternate titles.
- Publish-ready body/caption.
- Soft CTA.
- 6-12 hashtags.

Create `post-copy.md` as a separate paste-ready file when the user wants publication assets separated from the production notes.

## Title Rules

- Make the recommended title benefit-led.
- Align with the cover title, but do not force exact duplication.
- Keep it short enough for Xiaohongshu scanning.
- Use concrete outcomes, numbers, audience labels, or pain points when the source supports them.
- Do not promise unverifiable results.

Title candidate mix:

- Benefit: `{主题}：让{人群}少走弯路`
- Pain point: `{场景}别乱做，先看这份清单`
- List: `{数字} 步把{混乱状态}变成{可用结果}`
- Tutorial: `{工具/方法}上手：照这个流程走`
- Contrast: `别急着{错误动作}，先{正确动作}`
- Save-worthy: `{主题}看不懂？这张图讲清楚`

## Body Template

```markdown
# {推荐标题}

{一句话说明为什么值得收藏}

图里整理了：
1. {要点 1}
2. {要点 2}
3. {要点 3}

适合：
- {人群/场景 1}
- {人群/场景 2}

使用建议：
- {如何使用这套图}
- {下一步动作}

{软 CTA}

{hashtags}
```

## Tone

- Natural Chinese Xiaohongshu tone: practical, direct, save-worthy.
- Explain why the reader should save the post in the first sentence.
- Pull "图里整理了" bullets from actual card content.
- Name concrete readers or situations under "适合".
- Use a soft CTA such as saving, commenting with a use case, or asking for a teardown.
- Mix broad hashtags with specific topic tags.
